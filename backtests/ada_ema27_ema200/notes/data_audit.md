## 3. Загрузчик

`src/fetch_data.py` — два источника:

* `--source cache` (по умолчанию) — читает провалидированные CSV DATA-002;
* `--source api` — самостоятельная постраничная выгрузка Bybit V5 (`category=linear`,
  `limit<=1000`, курсор идёт вперёд по времени, ответ разворачивается из
  убывающего порядка, ретраи с экспоненциальным backoff).

Пишет `timestamp_utc, open, high, low, close, volume, turnover, ema27, ema200`.

**Прогрев EMA.** EMA считается на расширенном окне (`--warmup-bars`, по умолчанию
600 = 3×EMA200) и только потом обрезается до целевого окна — иначе первые ~200
баров EMA200 были бы смещены начальным значением.

**Проверка на выходе.** `validate()` падает, если сетка баров неполная,
есть дубликаты, NaN в OHLCV или не совпадают границы окна — молчаливые дыры
недопустимы.

**Кросс-проверка API против кэша** (1h, 2023-07-01..2023-07-03, 48 баров):
все шесть колонок OHLCV совпадают побитово (max |Δ| = 0.0), таймстемпы идентичны.
Пагинация и порядок сортировки корректны.

## 4. Известные ограничения

* **Формат вывода — CSV, не parquet.** В окружении нет ни `pyarrow`, ни
  `fastparquet`, а `pip` недоступен (`No module named pip`). Скрипт пытается
  писать parquet и при отсутствии движка откатывается на CSV с явным
  предупреждением в stderr. После `pip install pyarrow` тот же вызов начнёт
  писать parquet без изменений в коде.
* **EMA200 на 1d прогрета не полностью.** Источник начинается с 2023-01-01, т.е.
  до 2023-07-01 доступен всего 181 дневной бар при запрошенных 600. Скрипт
  печатает предупреждение. Для EMA200 на дневках данных не хватает —
  для стратегии использовать 15m/1h/4h, где прогрев полный.
* Сгенерированные файлы кладутся в `backtests/ada_ema27_ema200/data/` и
  исключены из git (`.gitignore`) как производные — воспроизводятся скриптом.

## 5. Решение

**Данные пригодны для бэктеста.** Отдельная выгрузка с биржи не требуется:
локальный кэш DATA-002 полон, хэш-верифицирован, покрывает окно
2023-07-01..2024-12-31 без единого гэпа на всех четырёх таймфреймах.

Рабочий путь: `python3 src/fetch_data.py --tf 1h` (источник — кэш).
`--source api` оставлен как проверенный резервный путь, если кэш недоступен.

Рекомендуемый ТФ для EMA27/EMA200 — **1h** (13 200 баров, полный прогрев EMA200);
15m и 4h также готовы.
