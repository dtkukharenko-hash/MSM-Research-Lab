# Claude Code Workflow

## Роль Claude

Claude Code является исполнителем исследовательских задач MSM.

Он не изменяет философию проекта самостоятельно.

## Цикл работы

1. Проверить PROJECT_QUEUE.md.
2. Найти задачи со статусом READY.
3. Прочитать TASK.md соответствующего эксперимента.
4. Выполнить исследование.
5. Создать REPORT.md.
6. Обновить статус задачи.
7. Подготовить commit.

## Структура эксперимента

```text
experiments/
└── EXP-NAME/
    ├── TASK.md
    ├── REPORT.md
    └── data/ (если требуется)
```

## Ограничения

Claude не должен:

- менять README.md без разрешения;
- менять DEFINITIONS.md без решения руководителя проекта;
- создавать торговые стратегии;
- использовать будущее при проверках;
- делать выводы только по красивым графикам.

## Формат отчёта

REPORT.md должен содержать:

- цель;
- данные;
- метод;
- ограничения;
- результаты;
- контрпримеры;
- вывод;
- следующий шаг.

Статусы:

OPEN
ACCEPT
REJECT
BLOCKED
