# EXP-003 — Minimal Object Review

## Статус

ACTIVE / REPORT_READY

## Контекст

EXP-002_PLATFORM_EXISTENCE завершён с вердиктом REJECT: площадка в текущем операциональном определении
не принимается как минимальный объект модели.

EXP-003 — методологическая ревизия после REJECT площадки. Это не запуск расчётов и не поиск торговой
стратегии.

## Цель

Пересмотреть кандидатов минимального объекта MSM после отклонения площадки и определить, какой кандидат
имеет смысл формализовать следующим.

## Кандидаты к пересмотру

- переход / марш;
- смена режима;
- направление / режим;
- импульс;
- строгая площадка;
- отсутствие локализованного объекта.

## Задача Claude

Провести методологический аудит каждого кандидата.

Для каждого кандидата:

1. дать краткое рабочее определение;
2. указать, является ли объект локализованным или режимным;
3. указать, что невозможно определить без будущего;
4. указать, какие контрпримеры ломают определение;
5. оценить риск повторения проблемы EXP-002: объект находится так же хорошо на null-рядах;
6. предложить, стоит ли кандидат переводить в следующий формальный эксперимент.

## Запрещено

- запускать Python;
- искать торговую стратегию;
- подбирать входы, выходы, прибыльность или сигналы;
- менять научные выводы без опоры на EXP-002 и существующие документы;
- изменять `docs/DEFINITIONS.md`;
- изменять любые файлы `Irobot`.

## Ожидаемый результат

Создать:

```text
experiments/EXP-003_MINIMAL_OBJECT_REVIEW/REPORT.md
```

REPORT.md должен содержать:

- сравнительную таблицу кандидатов;
- методологический вывод после REJECT площадки;
- рекомендацию, какой объект проверять следующим или почему локализованный минимальный объект пока не найден;
- список открытых вопросов для архитектора.
