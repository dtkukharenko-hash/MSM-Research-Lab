# EXP-004A — Movement Family Discovery

## Статус

ACTIVE / READY

## Контекст

В EXP-004 было найдено 50 сильных направленных движений ADA 2023 4H. H-MARCH-001 в строгом виде отклонена:
только 12/50 имели стартовую сильную свечу.

EXP-004A рассматривает оставшиеся 38 движений без стартовой сильной свечи.

## Цель

Понять, какие типы начала и окончания есть у этих 38 движений, не задавая классы заранее.

## Метод

1. Взять 38 случаев из EXP-004, где не было стартовой сильной свечи.
2. Не придумывать классы заранее.
3. Алгоритм группировки:
   - взять первое движение из 38; оно становится эталоном группы №1;
   - сравнить его с оставшимися 37;
   - для каждого случая определить: похоже на эталон или не похоже на эталон;
   - все похожие случаи собрать в группу №1;
   - описать группу №1: как начинается движение, как заканчивается движение, что общее у случаев, какие
     отличия внутри группы;
   - затем взять первое неиспользованное движение; оно становится эталоном группы №2;
   - повторять, пока все 38 случаев не будут распределены.

## Важно

- Не использовать слово "марш" как основание классификации.
- Не использовать ZigZag как доказательство.
- Не искать входы.
- Не искать прибыль.
- Не менять Irobot.
- Не менять `docs/DEFINITIONS.md`.
- Можно использовать числовые признаки только как помощь, но финальная группировка должна быть описательной.

## Ожидаемые артефакты

Создать:

```text
experiments/EXP-004_MARCH_FEATURES/EXP-004A_FAMILY_DISCOVERY/REPORT.md
experiments/EXP-004_MARCH_FEATURES/EXP-004A_FAMILY_DISCOVERY/artifacts/families.csv
experiments/EXP-004_MARCH_FEATURES/EXP-004A_FAMILY_DISCOVERY/artifacts/cases.csv
```

Для каждого случая в `cases.csv` указать:

- `case_id`;
- `start_time`;
- `end_time`;
- `direction`;
- `assigned_family`;
- `why_assigned`;
- `start_description`;
- `end_description`;
- `notes`.

Для каждой группы в `families.csv` указать:

- `family_id`;
- `prototype_case_id`;
- `count`;
- `start_pattern`;
- `end_pattern`;
- `common_features`;
- `internal_variations`;
- `confidence`.

## REPORT.md должен ответить

1. Сколько семейств получилось.
2. Какие семейства самые крупные.
3. Есть ли одиночные случаи.
4. Какие типы начала движения реально наблюдаются.
5. Какие типы окончания движения реально наблюдаются.
6. Есть ли семейство, которое можно считать кандидатом на "марш".
7. Что проверить следующим шагом.

Вердикт: OPEN / FAMILY_FOUND / NO_STABLE_FAMILIES.
