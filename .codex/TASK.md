R3 defects that must be corrected:

2. Internal phases were imported from the completed R2 attempt list instead of being generated by a continuing parent state machine.

`NO_PARENT -> ACTIVE_PARENT -> LOCAL_PRICE_REGIME -> OPTIONAL_JOINT_REGIME -> CONFIRMED_PARENT_RESOLUTION -> NO_PARENT`

## Core causal architecture

Implement one chronological event loop over raw 4H bars.

Primary states:

- `NO_PARENT`
- `PARENT_BUILDING_BOUNDS`
- `PARENT_ACTIVE_INSIDE`
- `LOCAL_PRICE_CANDIDATE`
- `ACTIVE_PRICE_OUTSIDE_REGIME`
- `ACTIVE_EMA_DEPARTURE_REGIME`
- `JOINT_PARENT_CANDIDATE`
- `PARENT_RESOLVED`

Only one parent may be active at a time.

Only one local price decision state may be active at a time.

An EMA regime may coexist with an active parent and with a price regime.

No state may inspect a bar after its current causal decision bar.

## Independent parent start

When state is `NO_PARENT`:

1. Scan raw bars chronologically for the next causal loss-of-agreement/core-trigger sequence using the validated R2 start logic.
2. Determine the candidate parent start using the last aligned run and causal loss-of-agreement logic.
3. Build robust initial upper and lower body boundaries from raw bars using the R2 causal estimators.
4. Confirm the lower reaction causally.
5. If bounds do not confirm, continue scanning raw bars; do not create a parent from an imported result.
6. Once confirmed, create a new parent and enter `PARENT_ACTIVE_INSIDE`.

After a confirmed parent resolution, start looking for the next parent only from the bar after parent-resolution confirmation.

Do not merge or split parents after the run.

## Parent price boundaries

Use the R2 body-based rules:

- upper boundary: median of the three highest causal `body_high` values from the aligned source interval;
- lower boundary: median of the three lowest causal `body_low` values through causal lower-reaction confirmation;
- wick extremes stored separately;
- no wick-only boundary update.

Boundaries remain frozen except after a causally confirmed accepted extension.

For every boundary version record:

- old upper/lower;
- proposed body level;
- new upper/lower;
- causal evidence interval;
- last timestamp used;
- ignored wick extreme;
- version number.

## Local price candidate

Use the validated R2 sequential attempt constants:

- `OUTSIDE_CLEARANCE_ATR = 0.15`
- `MIN_DECISION_BARS = 4`
- `MAX_DECISION_BARS = 12`
- `DEEP_RECLAIM_ATR = 0.15`
- `OUTSIDE_MAJORITY = 0.60`

Start an UP local candidate when:

`close > active_upper + 0.15 * ATR14`

Start a DOWN local candidate when:

`close < active_lower - 0.15 * ATR14`

Freeze active parent bounds and candidate ATR.

Process bar by bar and classify:

- `LOCAL_ACCEPTED_UP_DEPARTURE`
- `LOCAL_ACCEPTED_DOWN_DEPARTURE`
- `LOCAL_ACCEPTED_UP_EXTENSION`
- `LOCAL_ACCEPTED_DOWN_EXTENSION`
- `LOCAL_REJECTED_UP_EXCURSION`
- `LOCAL_REJECTED_DOWN_EXCURSION`

A local accepted departure never closes the parent by itself.

## Active price-outside regime

A local accepted departure creates an `ACTIVE_PRICE_OUTSIDE_REGIME`.

Required fields:

- regime ID;
- parent ID;
- direction;
- local candidate time;
- first outside close;
- local acceptance confirmation time;
- frozen parent boundary;
- frozen ATR;
- outside-close counts;
- deep-reclaim counts;
- current active/inactive status;
- termination reason and time.

The price regime remains active while the movement is still causally outside the frozen parent boundary.

Terminate the active price regime when one of these occurs:

1. `PRICE_RETURN_TERMINATION`: three consecutive closes deeply reclaim the frozen parent boundary.
2. `OPPOSITE_ACCEPTED_REGIME`: after the prior regime has terminated, an accepted local departure in the opposite direction is confirmed.
3. `PARENT_RESOLUTION`: a joint candidate confirms parent resolution.
4. `TRAIN_END`: development data ends.

Do not create repeated same-direction local candidates while the same price-outside regime remains active.

On `PRICE_RETURN_TERMINATION`, evaluate whether the completed outside shelf qualifies as an accepted parent extension using only bars from the original local candidate through the causal return-termination bar.

Accepted extension criteria:

- at least 3 closes outside the frozen boundary;
- at least 2 consecutive outside closes;
- median outside close at least `0.10 ATR14` beyond the frozen boundary using candidate ATR;
- price later reclaimed enough to terminate the outside regime;
- proposed boundary uses the median of up to three relevant outside body values;
- never use a wick as the boundary.

If criteria fail, preserve the existing parent boundary.

## EMA geometry as active regimes

R4 must use active EMA regimes rather than arbitrary event-age matching.

Implement two causal scales.

### Bootstrap EMA scale

Use only while fewer than 24 closed bars have elapsed since the active parent start:

- lookback: 12 prior closed bars;
- current bar excluded;
- compact-band thresholds from R2:
  - width `<= 0.60 ATR14`;
  - absolute net change `<= 0.35 ATR14`.

Label source scale:

`BOOTSTRAP_EMA12`

A bootstrap event may create a parent-resolution EMA regime only if its candidate and confirmation both occur after the active parent start.

### Parent EMA scale

Once at least 24 closed bars have elapsed since parent start, use:

- lookback: 24 prior closed bars;
- current bar excluded;
- band width `<= 0.90 ATR14`;
- absolute net change `<= 0.50 ATR14`;
- departure clearance `0.10 ATR14`;
- two consecutive bars beyond the frozen band edge for confirmation.

Label source scale:

`PARENT_EMA24`

After parent age reaches 24 bars, new bootstrap events cannot be created. An already active bootstrap regime may remain active until its causal termination.

### EMA regime classification

Qualified UP:

`EMA_REGIME_UP_AWAY_FROM_EMA200`

Requirements:

- confirmed upward departure;
- `ema_gap_change_6_atr > 0` at confirmation.

Qualified DOWN:

`EMA_REGIME_DOWN_TOWARD_EMA200`

Requirements:

- confirmed downward departure;
- `ema_gap_change_6_atr < 0` at confirmation.

Other confirmed departures remain diagnostics:

- `EMA_REGIME_UP_GAP_NOT_EXPANDING`
- `EMA_REGIME_DOWN_GAP_NOT_SHRINKING`

Only qualified regimes can participate in parent resolution.

### EMA regime lifecycle

After confirmation, create `ACTIVE_EMA_DEPARTURE_REGIME`.

The regime remains active until:

1. `EMA_RETURN_TERMINATION`: EMA27 is inside the frozen EMA band for two consecutive closed bars;
2. `PARENT_RESOLUTION`;
3. `TRAIN_END`;
4. after valid rearm, a new confirmed qualified regime supersedes it.

Record active duration and every bar-level active state.

The same EMA regime may participate in more than one distinct price/joint attempt after a failed price-based joint candidate, provided the EMA regime is still causally active.

Do not duplicate the EMA event itself.

## Exact EMA rearm

After a confirmed EMA departure, no new EMA candidate on the same scale may be created until either:

### RETURN_REARM

EMA27 is inside the previous frozen band for two consecutive closed bars.

### NEW_BAND_REARM

A newly compact prior window is available and every bar used by that window is strictly later than the previous confirmation bar.

For lookback `L`, if current candidate index is `t` and previous confirmation index is `c`, require:

`first_prior_window_index = t - L`

and:

`first_prior_window_index > c`

Do not substitute only `t - c >= L`.

Record suppressed duplicate candidates and prove that none becomes an EMA event before rearm.

## Joint parent-resolution candidate by regime overlap

Do not use an arbitrary “most recent event within N bars” rule.

Create `JOINT_PARENT_CANDIDATE` at the first bar where all are simultaneously true:

1. an active accepted price-outside regime exists;
2. an active qualified EMA-departure regime exists;
3. both regimes have the same direction;
4. both belong to the currently active parent;
5. neither regime has terminated;
6. no joint candidate is already active.

The joint candidate time is the first causal overlap bar.

Freeze:

- current parent upper/lower boundaries;
- price regime ID and original first outside close;
- EMA regime ID and frozen EMA band;
- direction;
- ATR14 at joint candidate creation;
- EMA gap at joint candidate creation.

Record price-regime age and EMA-regime age at overlap, but do not reject solely due to age while the regimes remain active.

## Joint persistence probation

Use:

- `JOINT_PROBATION_BARS = 12`
- minimum outside fraction `0.67`
- minimum EMA-beyond fraction `0.67`
- price deep-reclaim threshold `0.15 ATR14`

Start probation on the joint-overlap bar and count that bar as observed bar 1.

Evaluate sequentially. Stop at first causal success or failure.

Maintain:

- observed bars;
- price outside count/fraction;
- price deep-reclaim run;
- EMA beyond-edge count/fraction;
- EMA return-inside run;
- gap at candidate and current gap;
- last timestamp used.

### Early joint failure

Fail immediately on:

- three consecutive price deep-reclaim closes; or
- two consecutive EMA27 values inside the frozen EMA band, which also terminates the EMA regime.

### Joint confirmation

At observed bar 12 confirm only if all directional conditions hold.

UP:

- at least 8 of 12 closes above frozen parent upper boundary;
- final close above frozen parent upper boundary;
- no 3-bar price deep-reclaim run;
- at least 8 of 12 EMA27 values above frozen EMA upper edge;
- final EMA27 above frozen EMA upper edge;
- EMA27 above EMA200;
- final EMA gap greater than gap at joint creation.

DOWN:

- at least 8 of 12 closes below frozen parent lower boundary;
- final close below lower boundary or no more than `0.10 ATR14` above it during shallow retest;
- no 3-bar price deep-reclaim run;
- at least 8 of 12 EMA27 values below frozen EMA lower edge;
- final EMA27 below frozen EMA lower edge;
- final EMA gap smaller than gap at joint creation.

Do not require EMA200 to slope downward.

On confirmation:

- `CONFIRMED_PARENT_UP_RESOLUTION` or `CONFIRMED_PARENT_DOWN_RESOLUTION`;
- effective parent resolution = first outside close of the active price regime;
- local price confirmation = original local accepted-departure confirmation;
- joint overlap time stored separately;
- parent confirmation = 12th probation bar;
- parent closes only at confirmation.

## Required failed-joint continuation

This is a critical R4 requirement.

When a joint candidate fails:

1. Keep the same parent active.
2. Do not end parent bar-feature coverage.
3. If failure is caused by price deep reclaim:
   - terminate the active price regime;
   - evaluate accepted extension through the failure bar;
   - keep an EMA regime active if it has not independently terminated;
   - resume raw-bar local-candidate scanning from the next bar.
4. If failure is caused by EMA return while price remains outside:
   - terminate the EMA regime;
   - keep the active price regime alive if price has not reclaimed;
   - allow a later newly rearmed same-direction EMA regime to overlap the still-active price regime.
5. A later distinct local price regime may reuse the still-active same-direction EMA regime.
6. A second joint candidate must be possible after the first one fails.

Specifically verify on raw bars whether a new downside local/joint candidate appears after the early-January failed candidate. Do not hardcode January dates in detection.
